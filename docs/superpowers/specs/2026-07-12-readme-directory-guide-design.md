# Data_sheet README 目录说明设计

## 目标

为 `WQ-2001-mc/Data_sheet` 仓库增加一份简洁的中文 `README.md`，使访问者无需打开各级目录即可理解资料的分类方式和当前收录内容。

## 内容结构

README 由以下部分组成：

1. 仓库简介：说明本仓库用于分类保存芯片、FPGA 开发板、红外散斑投影器和相机相关资料手册。
2. 目录结构：使用代码块展示当前完整目录树，包括尚未收录文件的空目录。
3. 目录说明：用简短列表解释每个一级目录和相机二级分类的用途。
4. 当前资料：明确列出 `相机手册/结构光/gemini深度相机规格书.pdf`。
5. 维护说明：说明后续资料应放入对应分类目录；新增类别时同步更新 README。

## 目录范围

README 展示并解释以下结构：

```text
.
├── FPGA开发板手册/
├── 红外散斑投影器/
├── 芯片手册/
└── 相机手册/
    ├── IR相机/
    ├── tof/
    └── 结构光/
        └── gemini深度相机规格书.pdf
```

## Git 空目录处理

Git 不跟踪空目录。为使 GitHub 页面保留本地完整结构，需要在以下当前空目录中放置 `.gitkeep`：

- `FPGA开发板手册/`
- `红外散斑投影器/`
- `芯片手册/`
- `相机手册/IR相机/`
- `相机手册/tof/`

其中 `芯片手册/.gitkeep` 和 `相机手册/tof/.gitkeep` 已存在于仓库；本次只新增其余三个占位文件。

## 提交范围

最终功能提交只包含：

- 根目录 `README.md`
- `FPGA开发板手册/.gitkeep`
- `红外散斑投影器/.gitkeep`
- `相机手册/IR相机/.gitkeep`
- 本设计规格及后续实施计划文件

不修改现有 PDF，也不引入本地上层仓库的其他文件。

## 验证标准

- README 中的目录树与提交后的 Git 树一致。
- README 中引用的 PDF 路径真实存在。
- 本地 `main` 与远端 `main` 在推送后具有相同提交 SHA。
- 工作树在推送后保持干净。
